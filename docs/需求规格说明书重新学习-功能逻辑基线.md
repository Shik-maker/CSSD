# CSSD全自动化追溯系统需求规格说明书重新学习 - 功能逻辑基线

来源文档：`CSSD全自动化追溯系统_需求规格说明书_v1.0.docx`

本文档用于纠正此前实现偏差，并作为后续开发验收基线。系统不是单一 Web 页面原型，而是围绕同一套后端业务数据协同的三端系统：

- Web 管理后台：登录、基础信息、系统参数、各区域工作区、追溯查询、统计报表、权限管理。
- Windows 触摸操作台：安装在 Windows 触屏一体机，通过配置文件切换回收、清洗、配包、打包、灭菌、发放等工位。
- Android PDA：以 APK 安装到 PDA，支持下收、下送，断网离线缓存，回到医院内网 WiFi 后同步到服务器。

## 一、系统总目标

建设覆盖 `回收 -> 清洗 -> 配包 -> 打包 -> 灭菌 -> 发放 -> 使用/借包 -> 回收` 的复用无菌器械闭环追溯系统。

系统必须满足：

- 每个包、每次操作、每个批次、每台设备、每个操作人、每个筐、每段视频、每次异常、每次监测结果均可追溯。
- 触摸台/PDA 操作产生业务单据，Web 后台对应工作区必须能查询、查看、必要时编辑或补录。
- 后台基础信息与配置必须驱动触摸台/PDA 行为，而不是各端各自写死。
- 所有端通过同一套后端 API 和数据库互通。

## 二、终端组成与职责

### Web 管理后台

必须包含：

- 登录界面。
- 追溯查询。
- 统计报表。
- 基础信息管理：器械包、器械、包装、设备。
- 系统管理：参数配置、角色管理、人员管理、科室管理。
- 各工作区数据查看：回收区、清洗区、配包区、打包区、灭菌区、发放区、生物监测、PDA 同步、借包审核。
- 对由触摸台/PDA 产生的业务单据进行查看、补充、编辑、审核、留痕。

### Windows 触摸操作台

必须是 Windows 安装软件，不是 Web 页面伪装。

通过配置文件决定工位类型：

- 回收台。
- 清洗台。
- 配包台。
- 打包台。
- 灭菌台。
- 发放台。

触摸台特点：

- 大按钮、触屏优化、高频扫码操作。
- 回收、配包、打包、发放等环节按配置强制录像。
- 读取后台基础数据、参数配置、当前任务。
- 操作完成后生成业务单据和追溯事件，后台工作区实时可查。

### Android PDA

必须是 APK，安装到 Android PDA。

职责：

- PDA 下收：科室护士/下收人员扫码回收，断网时本地缓存。
- PDA 下送：触摸台生成下送任务，PDA 下载任务，断网时可执行已下载任务。
- 回到医院内网 WiFi 后自动/手动同步服务器。
- 上传时保留真实操作时间、上传时间、设备号、操作人、同步状态。

## 三、主流程与业务闭环

### 普通包闭环

发放 -> 科室使用 -> 回收（PDA 下收/科室直接送达） -> 清洗 -> 配包 -> 打包 -> 灭菌 -> 发放。

### 借包闭环

科室申领 -> 审核 -> 发放 -> 科室使用 -> 归还（回收时自动识别并还包） -> 清洗 -> 配包 -> 打包 -> 灭菌 -> 发放。

### 关键状态原则

- 包状态是后端权威状态。
- 触摸台/PDA 不能绕过后端校验直接决定最终业务状态。
- 单据状态、包状态、批次状态、筐状态必须联动。
- 后台工作区显示的业务单据必须来自触摸台/PDA 实际操作。

## 四、回收模块

### 数据来源

- PDA 下收。
- 科室直接送达。
- 借包归还。

### 触摸台模式一：PDA 清点

流程：

- PDA 下收数据上传后，回收台出现红点数字和弹窗提醒。
- 操作员进入 PDA 清点，按下收人批次展示。
- 开始录像。
- 展示批次所有包列表和内容物汇总。
- 逐包清点核对。
- 可登记异常：缺失、破损、血渍、数量不符、标识不清等。
- 结束录像，数据进入待清洗。

后台要求：

- Web 回收区能按 PDA 批次查看清点任务。
- 能查看下收人、批次、包清单、异常、备注、视频、清点状态。
- 清点完成后生成回收单。

### 触摸台模式二：科室回收

流程：

- 操作员选择科室或扫描包条码自动识别科室。
- 开始录像。
- 逐包扫码加入回收列表。
- 支持标记加急、污染、特殊感染、破损等。
- 回收过程中可扫描空闲筐条码将已回收包装筐。
- 支持拖拽调整包归属、出入筐和排序。
- 结束录像，进入待清洗。

后台要求：

- Web 回收区能查看和编辑回收单。
- 回收单必须包含总回收数、还包数、来源、回收人、回收时间、科室、包清单、异常、备注、关联筐、视频。
- 编辑要留痕，并影响后续清洗待办。

### 筐管理

这是独立业务能力，不能遗漏。

筐状态：

- 空闲：未被使用。
- 待清洗：已绑定包，进入清洗端待清洗列表。
- 清洗中：已入锅。
- 已清洗：已出锅待释放。
- 空闲：清洗出锅后自动释放。

业务规则：

- 筐不设容量上限。
- 一个回收批次可绑定多个筐。
- 一个筐可绑定多个包。
- 入锅时可选择部分或全部入锅。
- 清洗出锅后筐自动释放为空闲。
- 已入筐的包是否允许移出由配置控制。
- 后台必须有筐管理/筐状态查看能力。
- 回收区、清洗区必须能围绕筐查看包清单和状态。

### 借包归还自动识别

规则：

- 回收扫码时系统自动识别“已借出”状态的包。
- 列表显示：包名 + 回收 X 件 + 还包 X 件。
- 还包数单独统计，不计入待发放任务。
- 自动更新借包单状态为“已归还”。
- 待发放任务数量 = 回收数 - 还包数。

后台要求：

- 回收单必须展示还包数。
- 借包单必须关联回收记录。
- 借包归还不可简单当作普通回收。

## 五、清洗模块

### 数据来源

回收完成后自动推送，待清洗列表按筐展示。

### 操作流程

- 待清洗列表按筐展示：筐编码、包数、来源科室。
- 操作员选择清洗设备，设备列表只显示空闲清洗机。
- 选择设备预设程序，可为空。
- 装载支持扫筐入锅、全部入锅。
- 扫筐入锅弹窗显示筐内包清单，支持部分入锅。
- 修改入锅数量时包清单勾选自动联动。
- 出锅前已入锅清单可修改清洗单。
- 开始清洗后记录设备、程序、开始时间。
- 设备运行参数实时采集并记录。
- 完成后人工确认：合格则出锅，筐释放为空闲，包进入待配包；不合格则整锅或单包重新入锅/换锅清洗。

后台要求：

- Web 清洗区能查看清洗记录、批次号、设备、程序、筐列表、包列表、开始/结束时间、参数、结果。
- 能按配置支持整锅退回、单包退回或挂起。
- 能查看清洗方式是否被临时覆盖及覆盖留痕。

### 清洗类型管理

- 器械包基础信息中预设机洗/手洗。
- 待清洗列表支持机洗/手洗筛选。
- 清洗端支持临时覆盖清洗方式，并留痕。
- 允许覆盖角色由配置控制。

### 并发控制

- 点击包名弹窗时对包或筐加锁。
- 其他人操作提示“正在被 XX 操作中”。
- 锁超时自动释放，默认 120 秒可配置。
- 提交时后端做原子扣减校验。
- 后台可查看锁信息，管理员可手动释放。

## 六、配包模块

### 数据来源

清洗完成后自动推送，以包为单位。

### 操作流程

- 配包台自动刷新待配包列表。
- 操作员点击待配包后开始录像。
- 屏幕展示包内容物清单和配包标准图谱。
- 操作员逐一核对器械。
- 配包完成后结束录像。
- 数据推送到打包台。

后台要求：

- Web 配包区能查看待配包任务、已配包记录、配包人、配包时间、包内容物、图谱、视频。
- 后台基础信息中的器械包内容物清单和配包图谱必须驱动触摸台展示。

## 七、打包模块

### 数据来源

配包完成后自动推送。

### 操作流程

- 打包台自动刷新待打包包。
- 操作员点击任务后开始录像。
- 展示内容物清单和打包标准图谱。
- 操作员封包、贴标签。
- 标签包含包名、包内物、配包人、打包人、失效时间、打印时间、灭菌时间等字段。
- 打包完成后结束录像。
- 数据推送灭菌端。

后台要求：

- Web 打包区能查看待打包任务、已打包记录、配包人、打包人、标签信息、视频。
- 打包完成后必须触发标签打印能力。
- 标签模板和打印行为必须配置化，并由后台打印模板管理维护。

### 打包标签内容规则

器械包标签至少包含：

- 第一行显示包名。
- 显示包内物清单。
- 显示配包人：取配包环节登录并完成配包操作的用户。
- 显示打包人：取打包环节登录并完成打包操作的用户。
- 显示失效时间：根据器械包管理中选择的包材有效期计算，不应手工随意填写。
- 显示打印时间。
- 显示灭菌时间：灭菌完成后应能补打或打印包含灭菌时间的标签；如打包时预打印，则灭菌时间字段需按业务规则处理为空、预留或后续补打。

失效时间计算规则：

- 器械包管理中选择包材。
- 包材管理维护有效期天数。
- 包灭菌合格后，以灭菌时间为基准，加上包材有效期天数，得到失效时间。
- 如果业务要求打包完成即打印标签，则标签上的失效时间依赖后续灭菌时间，必须设计为“灭菌后打印/补打”或“预打印不含失效时间/灭菌时间”的可配置策略。

### 打印模板管理

后台必须增加打印模板管理功能。

能力要求：

- 支持通过设计器配置标签或单据模板。
- 可以选择打印字段。
- 可以配置字段属性，例如显示名称、位置、字号、加粗、对齐、是否显示、条码/二维码类型等。
- 模板应版本化或至少记录修改人、修改时间，避免已打印历史无法追溯。
- 打印记录必须留痕，包括模板、打印人、打印时间、打印对象、打印次数。

模板类型至少包括：

- 器械包标签打印模板。
- 器械包唯一码标签打印模板。
- 借包单打印模板。
- 发放单打印模板。

后续数据模型需补充：

- 打印模板表。
- 打印模板字段配置表或 JSON 配置。
- 打印记录表。
- 标签打印任务表。
- 包材有效期与器械包实例失效时间关联字段。

## 八、灭菌模块

### 数据来源

打包完成后自动推送，以包为单位。

### 操作流程

- 灭菌端待灭菌列表按包展示。
- 操作员选择灭菌设备，只显示空闲且符合灭菌类型的设备。
- 选择设备预设程序，可为空。
- 装载支持扫码装载、批量装载、全部装载、按灭菌方式装载。
- 包灭菌方式与设备类型不匹配时，默认禁止装载。
- 生物监测默认跟随设备配置，可由操作员切换。
- 开始灭菌后记录设备、程序、开始时间、锅号/锅次。
- 设备运行参数实时采集。
- 灭菌完成后进行物理、化学、生物三项监测判定。
- 全部合格后包状态为已灭菌，推送发放端。

后台要求：

- Web 灭菌区能查看锅次、设备、程序、包清单、物理参数、化学结果、生物状态、开始/结束时间、结果。
- 能处理不合格：重新灭菌、换锅灭菌、报损。

### 监测判定

- 物理监测：系统自动判定 + 人工确认，基于温度、压力、时间。
- 化学监测：人工登记，检查化学指示卡。
- 生物监测：独立模块录入，与灭菌锅次关联。

### 不合格处理

- 物理不合格：重新灭菌、换锅灭菌、报损。
- 化学不合格：重新灭菌、换锅灭菌、报损。
- 生物不合格：按配置召回自上次合格以来所有批次或仅当前批次。

## 九、生物监测模块

### 定位

独立于灭菌流程的并行模块，与灭菌锅次关联。

### 操作流程

- 灭菌入锅时选择是否需要生物监测。
- 灭菌完成后，生物监测界面出现待录入记录。
- 操作员录入结果，可自动读取或手动录入。
- 合格后，对应锅次的包允许发放。
- 不合格后，自动锁定相关批次，生成召回清单。
- 未录入或不合格，包禁止发放。

后台要求：

- Web 生物监测区必须有待录入、已录入、不合格召回列表。
- 发放逻辑必须校验生物监测状态。
- 生物阳性自动锁定是否启用由配置控制。

## 十、发放模块

### 三种模式

- 科室发放：按回收单发放，由触摸台操作员执行。
- PDA 发放：触摸台生成下送任务，下送人 PDA 执行。
- 借包审核：处理科室申领。

### 科室发放

流程：

- 左侧显示科室和待发放回收单。
- 点击回收单显示待发包清单。
- 操作员逐包扫码发放。
- 可少发，绝对不能多发。
- 完成后自动打印发放单，是否自动由配置控制。
- 待发放任务数量 = 回收数 - 还包数。

后台要求：

- Web 发放区能按科室、回收单查看待发放任务。
- 能查看已发放、部分发放、未发放状态。
- 不能多发必须由后端校验。

### PDA 发放

触摸台端：

- 显示 PDA 下收回来的待发放包，按 PDA 下收批次聚合。
- 操作员选择下送人。
- 确认交接后生成下送任务推送 PDA。
- 可打印交接单。

PDA 端：

- 下送人登录 PDA，同步下送任务。
- 到科室后扫描科室条码或选择科室。
- 逐包扫码发送，可少不可多。
- 科室护士 PDA 签字确认可配置。
- 完成配送后数据上传。
- 离线时，已下载任务可正常执行，回 CSSD 后上传。

后台要求：

- Web 发放区/PDA 工作区能查看下送任务、下送人、科室确认、签字、同步状态。

### 借包审核

流程：

- 科室提交申领单后，发放台借包审核入口显示红点和弹窗。
- 操作员审核通过或驳回。
- 驳回必须填写原因。
- 通过后科室端弹窗提示“请取货”。
- 通过后进入借包列表，状态待借/待取货。
- 科室派人到窗口取货。
- 操作员扫码发放，支持多单合并取货。
- 可打印借包单。
- 科室无需扫码确认，发放即视为取走。

借包单状态：

待审核 -> 已驳回 / 待取货 -> 已借出 -> 已归还。

后台要求：

- Web 借包审核区必须可审核、驳回、填写原因、查看取货/借出/归还状态。
- 借包发放和普通发放要区分。

## 十一、科室端操作

访问方式：

- Web 端 + 小程序，或按配置仅 Web/仅小程序。

功能：

- 借包申请。
- 申领单状态查看：待审核、已驳回、待取货、已借出。
- 审核结果通知。
- 包状态查询。
- 科室在库包列表。
- 近效期提醒。
- 退回申请。
- 使用登记，包使用后与患者关联。
- 科室统计：申领量、使用量、退回量、库存概览。
- 消息通知：审核结果、发放通知、近效期提醒、系统公告。

后台要求：

- 科室端产生的借包、使用、退回、库存状态必须进入后台管理端。
- 患者关联属于追溯链的一部分，后续数据模型不能遗漏。

## 十二、后台管理配置体系

### 菜单结构

- 追溯查询。
- 统计报表。
- 基础信息：
  - 器械包管理。
  - 器械管理。
  - 包装管理。
  - 设备管理。
- 系统管理：
  - 参数配置。
  - 角色管理。
  - 人员管理。
  - 科室管理。

### 基础信息 CRUD

必须支持增删改查，而不是只读展示。

器械包管理：

- 包编码、名称、分类、灭菌方式、清洗方式、有效期、关联包装、内容物清单、配包图谱、打包图谱、是否启用。

器械管理：

- 器械编码、名称、规格型号、分类、单位。

包装管理：

- 包装名称、有效期天数、是否启用。

设备管理：

- 清洗机：设备编号、名称、型号、位置、状态、预设程序列表。
- 灭菌器：设备编号、名称、类型、型号、位置、状态、预设程序列表、是否默认生物监测。

系统管理：

- 参数配置按模块分类展示。
- 角色权限含菜单权限和按钮权限。
- 人员含工号、姓名、科室、角色、人员类型、状态、登录方式。
- 科室含编码、名称、类型、科室条码、是否启用。

### 默认角色

- 系统管理员：全部菜单，全部工位。
- 护士长：全部菜单但不含角色/人员/科室管理，全部工位。
- CSSD 操作员：追溯查询、统计报表、基础信息只读，全部工位。
- 下送人：追溯查询/统计报表查看，PDA 下送任务。
- 临床护士：追溯查询/统计报表查看，仅科室端。

- 配置变更留痕表。

## 十六、技术与部署约束

- Web 后台和触摸台运行于 Windows 环境。
- PDA 运行于 Android。
- 全系统部署在医院内网。
- 固定端断网禁止关键业务提交。
- PDA 是离线例外：无网络时缓存允许范围内的任务和扫码记录，回内网 WiFi 后上传。
- 实时通信用于配置推送、大屏刷新、工作区提醒。
- 视频上传异步压缩上传，不阻塞业务操作。
- 配置修改需完整留痕。

## 十七、后续开发验收原则

每个模块必须同时满足：

- 触摸台/PDA 能执行现场操作。
- 后端生成真实业务单据和明细。
- Web 后台对应工作区能看到单据。
- Web 后台能按权限查看、编辑、审核或补录。
- 单据变化能联动包状态、筐状态、设备状态、批次状态。
- 追溯查询能串起完整链路。
- 统计报表能按单据和明细聚合。
- 配置项能实际影响业务，而不是只显示。

## 十八、用户补充后的追溯与标签规则

### 1. 标签灭菌时间与失效时间

- 打包完成后必须打印器械包标签。
- 标签第一行显示包名。
- 标签必须显示包内物、配包人、打包人、失效时间、打印时间、灭菌时间。
- 配包人取配包环节登录并完成配包操作的用户。
- 打包人取打包环节登录并完成打包操作的用户。
- 灭菌时间按本次标签打印日期取值，精确到天。
- 失效时间按器械包选择的包材有效期计算：打印日期加包材有效期天数。
- 打印模板管理必须支持器械包标签、器械包唯一码标签、借包单、发放单四类模板。
- 模板设计器至少支持字段选择、字段显示名、位置、字号、加粗、对齐、显隐、条码或二维码属性。

### 2. 两种器械包追溯模式

- 手术室包采用一包一码模式。比如手术室有 10 个口护包，系统应建 口护包1 到 口护包10，每个包都有唯一编码，从回收开始即可扫唯一码追溯。
- 普通临床科室包采用批量模式。回收时没有打包后标签，不能依赖标签追溯回收环节。
- 临床批量包回收时必须生成回收单和追溯批次号，批次号贯穿回收、清洗、配包、打包。
- 临床批量包打包打印标签时，生成的每一张标签必须关联来源批次。后续扫标签追溯时，既能看到打包后的标签链路，也能反查标签生成前的回收批次和回收单。
- 批次要记录总数量、剩余数量、来源科室、来源回收单、绑定筐、当前状态、操作事件。
- 批量包允许部分打包、部分打印标签，剩余数量必须继续留在批次账本中。
- 所有单据、批次、标签、模板修改和后台编辑都必须留操作日志和数据审计日志。
